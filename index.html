About

I'm a Senior Cloud Platform Engineer with 10+ years across AWS, Azure, and GCP. For the past four years at Ellucian, I've been the primary technical owner of the AWS multi-account provisioning and lifecycle platform that the company's higher-education customer environments run on.

That means Control Tower, Account Factory, region enablement, disaster recovery architecture, cost and compliance automation — and increasingly, building the internal tools that let other engineers self-serve against the platform instead of filing a request to me directly.

Before Ellucian, I spent time inside AWS itself — first in Support, then in Professional Services — which is where most of my instincts about how customers actually use this stuff in production came from.

Selected work

Case studies from Ellucian's AWS platform, plus one project with code on GitHub.

Case study — internal platform tool

Self-service account provisioning

Other engineers across the org needed new AWS accounts constantly — new colleges onboarding, additional accounts for existing ones — and every request landed on me. I built a self-service tool, solo, that lets engineers provision accounts themselves against the same automation I'd been running by hand.

Result: removed a standing bottleneck from the provisioning workflow. Now in active use org-wide.

Case study — agentic infrastructure tooling

MCP server for Control Tower & Account Factory

Investigating account or org-state issues during an incident or audit usually means jumping between consoles and CLI calls. I built an MCP server, solo, that exposes Control Tower and Account Factory state to natural-language queries — ask what's happening with an account, get an answer.

Result: faster investigation during incidents and audits, without needing to know the underlying API surface.

Case study — identity & access

Slack-based self-service access requests

Built, solo, a Slack-native tool that integrates with Okta to verify identity, then provisions SSO access via AWS IAM Identity Center — so engineers can request and receive account access without leaving Slack or filing a ticket.

Result: access requests resolved in Slack, identity-verified end to end.
